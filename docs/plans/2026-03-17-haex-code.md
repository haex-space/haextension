# haex-code - IDE Extension für haex-vault

## Übersicht

Code-Editor und Terminal-Extension für haex-vault. Basiert auf Monaco Editor und xterm.js, nativ integriert über vault-sdk. Kein VS Code Extension-Kompatibilität - stattdessen eigenständige, leichtgewichtige IDE.

**Kernprinzip:** haex-vault IST das Backend. Die IDE ist "nur" das Frontend. Shell-Befehle, Filesystem-Zugriff und alles weitere läuft über vault-sdk → Tauri invoke → Rust Backend.

## Tech-Stack

- **Framework:** Nuxt 4 SPA (wie alle Extensions)
- **Editor:** Monaco Editor (VS Code's Editor-Engine)
- **Terminal:** xterm.js
- **Filesystem:** vault-sdk Filesystem API (existiert bereits)
- **Shell:** vault-sdk Shell API (neu zu implementieren)
- **UI:** haex-ui Layer (shadcn-vue, Reka UI, Tailwind)
- **Layout:** Splitpanes für resizable Panels
- **Dateisuche:** fuse.js (clientseitig, mit .gitignore-Filtering)
- **State:** Kein persistenter State - jede Session startet frisch

## Datenflüsse

| Aktion | Pfad |
|--------|------|
| Datei öffnen | File Explorer → sdk.filesystem.readFile → Monaco Tab |
| Datei speichern | Monaco → sdk.filesystem.writeFile → Disk |
| Terminal Input | xterm.js keystroke → sdk.shell.write → PTY stdin |
| Terminal Output | PTY stdout → sdk.shell.onData event → xterm.js render |
| Terminal Resize | xterm.js resize → sdk.shell.resize → PTY SIGWINCH |
| Workspace öffnen | Dialog → sdk.filesystem.selectFolder → File Tree laden |
| Git Status | File Watcher events → Datei-Icons im Tree aktualisieren |

## Shell-Architektur

### Desktop (Tauri)
- PTY via `portable-pty` crate im Rust-Backend
- Vollwertiges interaktives Terminal (vim, htop, ssh, docker - alles funktioniert)
- Tauri Commands: create, write, resize, close
- stdout/stderr Streaming via Tauri Events

### Android (Tauri Mobile)
- `russh` crate für SSH-Sessions im Rust-Backend
- Gleiche SDK-API wie Desktop (`ShellSession`)
- Kein lokales Terminal möglich, nur Remote via SSH

### SDK Shell API (neu)

```typescript
sdk.shell.create(options: ShellCreateOptions): Promise<string>  // → SessionID
sdk.shell.write(sessionId: string, data: string): Promise<void>
sdk.shell.resize(sessionId: string, cols: number, rows: number): Promise<void>
sdk.shell.close(sessionId: string): Promise<void>
sdk.shell.onData(sessionId: string, callback: (data: string) => void): void
```

### Tauri Commands (neu)

```rust
extension_shell_create(shell?: String, cwd?: String) -> String  // SessionID
extension_shell_write(session_id: String, data: String) -> ()
extension_shell_resize(session_id: String, cols: u32, rows: u32) -> ()
extension_shell_close(session_id: String) -> ()
// Event: "shell:data:{session_id}" für stdout streaming
```

## UI Layout

### Desktop
- VS Code Layout: Sidebar links (File Explorer), Editor oben, Terminal unten
- Alle Panels resizable via Splitpanes
- Statusbar am unteren Rand

### Mobile (Android)
- Editor im Vollbild als Hauptansicht
- File Explorer als Overlay-Drawer von links (Swipe/Button)
- Terminal als Bottom-Sheet (hochziehbar)

## Features v1.0

### Terminal
- xterm.js mit PTY-Backend (Desktop) / russh (Android)
- Multi-Terminal Tabs (mehrere Sessions parallel)
- Interaktiv: vim, ssh, docker, htop - alles

### File Explorer
- Baumansicht mit lazy-loading pro Ordner
- Neue Datei / Neuer Ordner
- Umbenennen, Löschen
- Datei-Icons nach Typ
- Workspace öffnen via selectFolder Dialog

### Monaco Editor
- Syntax-Highlighting (30+ built-in Sprachen)
- Multi-File Tabs
- Eigene Theme-Auswahl (unabhängig vom System-Theme)
- Standard-Keybindings (Ctrl+S, Ctrl+Z etc.)

### Split Panels
- Editor + Terminal nebeneinander/übereinander
- Resizable via Drag

### Git Integration (minimal)
- Aktueller Branch in Statusbar
- Geänderte Dateien im File Explorer markiert
- Alles weitere über Terminal

### Dateisuche
- Fuzzy-Finder (Ctrl+P) mit fuse.js
- .gitignore-Filtering
- Clientseitig

### Settings
- In Datenbank gespeichert (eigene Tabelle)
- Font-Size, Editor-Theme, Default-Shell
- Kein Workspace-State-Persistenz (bewusst)

### Docker Integration
- Nur über Terminal in v1.0
- UI-Dashboard in späterer Version

## Offene Punkte / Follow-up

- **Shell-Permission Warnung:** Bei Installation von Extensions mit Shell-Permission sollte haex-vault explizit auf die Gefahr hinweisen (sudo, rm -rf etc.)
- **Android SSH:** russh-Integration als separates Teilprojekt nach Desktop-PTY
- **Docker UI:** Dashboard für Container-Management in v1.1+
- **Git UI:** Staged/Unstaged, Diff-View, Commit-UI in v1.1+
- **File Explorer erweitert:** Drag&Drop, Kontextmenü, Vorschau in v1.1+
- **Workspace-Persistenz:** Optional nachrüstbar (letzte Tabs, CWD etc.)
